## 1. Tree Navigator

Think `lazygit` or `k9s` style — a panel-based view for quickly moving through the work tree.

```
┌─ BackendService (Epic) ──────────────────────────────────────────────┐
│  ▶ Release Management (Deliverable) [s]          3/7 tasks done  │
│    ├─ Create Release Templates (Task) [p]                        │
│    ├─ Create Basic Guidance (Task) [p]                           │
│    └─ Release Mgmt Task TBD (Task) [p]                          │
│  ▼ Health & Synthetics (Deliverable) [s]         1/3 tasks done  │
│    ├─● Implement health endpoints (Task) [s]     ← active       │
│    ├─ Implement validation bootstrap (Task) [s]                  │
│    └─ Implement V1 Synthetics (Task) [p]                         │
│  ▶ Cleanup (Deliverable) [s]                     0/4 tasks done  │
│  ▶ Specs & Docs (Deliverable) [s]                0/2 tasks done  │
├──────────────────────────────────────────────────────────────────┤
│ [enter] expand/collapse  [space] checkout  [s] set state         │
│ [/] search  [n] new stub  [q] quit                               │
└──────────────────────────────────────────────────────────────────┘
```

Key interactions:
- **Arrow keys / j/k** — move through tree
- **Enter** — expand/collapse node
- **Space** — checkout (set as active context)
- **s** then shorthand — change state inline
- **n** — create stub as child of selected item
- **/** — fuzzy search across visible items
- **Tab** — switch to detail panel (right split?)
